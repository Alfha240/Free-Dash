# üì• GitHub Installation Guide

## üöÄ One-Command Installation

Users can install Free Dash directly from GitHub with a single command:

```bash
curl -fsSL https://raw.githubusercontent.com/Alfha240/Free-Dash/main/install.sh | bash
```

## üìã Installation Process

The `install.sh` script will:

1. ‚úÖ **Check Prerequisites**
   - Verify/Install Docker
   - Verify/Install Docker Compose
   - Check system requirements

2. ‚úÖ **Clone Repository**
   - Clone from GitHub: `https://github.com/Alfha240/Free-Dash.git`
   - Or update if already exists

3. ‚úÖ **Domain Configuration**
   - Prompt for main domain (e.g., `yourdomain.com`)
   - Prompt for API subdomain (e.g., `api.yourdomain.com`)
   - Auto-configure URLs

4. ‚úÖ **Generate Secrets**
   - Auto-generate JWT secrets
   - Auto-generate MongoDB password
   - Auto-generate Redis password

5. ‚úÖ **Pterodactyl Setup**
   - Prompt for Pterodactyl URL
   - Prompt for Pterodactyl API Key

6. ‚úÖ **Discord OAuth** (Optional)
   - Prompt for Discord credentials

7. ‚úÖ **Build & Start**
   - Build all Docker images
   - Start all services
   - Verify health checks

8. ‚úÖ **Domain Setup** (If domain provided)
   - Install Nginx
   - Configure reverse proxy
   - Setup SSL with Let's Encrypt

## üåê Domain Configuration

### During Installation

The script will ask:
- **Main Domain**: `yourdomain.com`
- **API Subdomain**: `api.yourdomain.com` (default: `api.yourdomain.com`)

### After Installation

If you skipped domain setup, run:

```bash
sudo ./setup-domain.sh
```

## üìù Manual Installation Steps

If you prefer manual installation:

```bash
# 1. Clone repository
git clone https://github.com/Alfha240/Free-Dash.git
cd Free-Dash

# 2. Copy environment template
cp .env.example .env

# 3. Edit .env file
nano .env

# 4. Build and start
docker-compose build
docker-compose up -d
```

## üîß Configuration

### Environment Variables (.env)

The installation script creates a `.env` file with:

```env
# Domain Configuration
FRONTEND_URL=https://yourdomain.com
VITE_API_URL=https://api.yourdomain.com
MAIN_DOMAIN=yourdomain.com
API_DOMAIN=api.yourdomain.com

# Auto-generated Secrets
JWT_SECRET=<auto-generated>
JWT_REFRESH_SECRET=<auto-generated>
MONGO_ROOT_PASSWORD=<auto-generated>
REDIS_PASSWORD=<auto-generated>

# Pterodactyl
PTERODACTYL_URL=https://panel.yourdomain.com
PTERODACTYL_API_KEY=ptla_xxxxx

# Discord (Optional)
DISCORD_CLIENT_ID=xxxxx
DISCORD_CLIENT_SECRET=xxxxx
```

### Domain DNS Setup

Before running domain setup, configure DNS:

```
A Record: yourdomain.com ‚Üí YOUR_SERVER_IP
A Record: api.yourdomain.com ‚Üí YOUR_SERVER_IP
```

1. **DNS Configuration**: Point your domain DNS to server IP before SSL setup
2. **Firewall**: Allow ports 80, 443, and optionally 3000
3. **Secrets**: Keep your `.env` file secure - it contains sensitive data
4. **Backups**: Set up regular MongoDB backups
5. **Updates**: Pull latest changes before rebuilding

## üÜò Troubleshooting

### Installation Fails

```bash
# Check Docker
docker --version
docker-compose --version

# Check logs
docker-compose logs
```

### Domain Not Working

1. Verify DNS: `dig yourdomain.com`
2. Check Nginx: `sudo nginx -t`
3. Check SSL: `sudo certbot certificates`

### Services Won't Start

```bash
# Check logs
docker-compose logs -f

# Restart services
docker-compose down
docker-compose up -d
```
